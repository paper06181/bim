# 1. 프로젝트 개요

## 1.1 무엇을 하는 시스템인가?

> **"BIM을 도입하면 얼마나 효과가 있을까?"**
> 이 질문에 **정량적으로 답변**하는 시뮬레이션 시스템

### 간단히 말하면:
```
건설 프로젝트 정보 입력
         ↓
시뮬레이션 실행 (BIM OFF vs ON)
         ↓
결과 비교: 공기/비용/품질
         ↓
의사결정 지원
```

## 1.2 왜 필요한가?

### 현재 문제점:
- ❌ BIM 도입 비용만 보임 (500만원~5000만원)
- ❌ 효과는 막연함 ("좋다더라...")
- ❌ 투자 의사결정 어려움

### 이 시스템의 해결:
- ✅ 정량적 효과 측정 (공기 30일 단축, 비용 3억 절감)
- ✅ 프로젝트별 맞춤 분석
- ✅ 명확한 ROI 계산

# 2. 시스템 작동 원리

## 2.1 전체 흐름 (아주 단순하게)

```
[1단계] 프로젝트 설정
 - 건물 정보: 연면적 883㎡, 예산 20억, 공기 12개월
 - BIM 품질: 우수/양호/보통/미흡 선택

[2단계] 시뮬레이션 실행
 - 설계 → 입찰 → 시공 → 준공 (4단계)
 - 각 단계마다 이슈 발생 (확률적)
 - 5개 에이전트가 회의해서 해결

[3단계] 결과 계산
 - BIM OFF: 지연 45일, 예산 초과 22%
 - BIM ON: 지연 15일, 예산 초과 8%
 - 차이: 30일 단축, 14% 절감

[4단계] 결과 출력
 - 텍스트 리포트 (txt)
 - 그래프 4종 (png)
 - 회의록 상세 (txt)
```

## 2.2 핵심 메커니즘

### 이슈 발생 시스템
```
이슈 카드 27개 준비
  ↓
매일 발생 확률 체크 (0.8~1.5%)
  ↓
발생하면 → 회의 소집
```

**예시**:
- 이슈: "설비-구조 간섭 미발견"
- 발생 확률: 1.5% / 일
- 영향: 지연 6~10주, 비용 2.5~4.0%

### 회의 시스템 (5개 에이전트)
```
[초기 회의] 문제 인식
  시공사: "현장에서 충돌 발견했습니다"
  설계사: "도면을 재검토하겠습니다"
  감리사: "계약 조건 확인하겠습니다"
  금융사: "리스크를 평가하겠습니다"
  건축주: "비용과 일정이 얼마나 영향 받습니까?"

[의사결정 회의] 해결 방안 결정
  설계사: "2주 내에 도면 수정 가능합니다"
  시공사: "작업 조정하면 3주 소요됩니다"
  감리사: "2.5주가 적정합니다"
  건축주: "2주로 진행하십시오" (최종 결정)
```

### BIM 효과 계산
```
BIM OFF (전통 방식)
  ↓
이슈 발생 → 시공 중 발견 → 큰 지연/비용
  예: 8주 지연, 3.5% 비용 증가

BIM ON (BIM 적용)
  ↓
이슈 발생 → 설계 단계 조기 탐지 → 작은 지연/비용
  예: 1.5주 지연, 0.3% 비용 증가

효과: 6.5주 절감, 3.2% 절감
```

# 3. 세팅 구조 상세

## 3.1 프로젝트 기본 세팅

### 📋 파일 위치: `config/project_config.py`

```python
# 기본 정보
PROJECT_NAME = "건설 프로젝트"
LOCATION = "프로젝트 위치"
USE = "건축물 용도"

# 규모
GFA = 883.5  # 연면적 (㎡)
TOTAL_BUDGET = 2_030_000_000  # 예산 (20.3억)
TOTAL_DURATION = 360  # 공기 (360일 = 12개월)

# 단계별 기간
PHASE_DURATIONS = {
    '설계': 90일,    # 3개월
    '입찰': 20일,    # 3주
    '시공': 300일,   # 10개월
    '준공': 20일     # 3주
}

# 금융 정보
PF_RATIO = 0.7  # PF 대출 70%
BASE_INTEREST_RATE = 0.055  # 연 5.5%
```

**왜 이렇게 설정했나?**
- 연면적 883㎡ = 중소형 근린생활시설 평균
- 예산 20억 = ㎡당 230만원 (서울 평균)
- 공기 12개월 = 소형 건물 평균
- PF 70% = 건설 프로젝트 일반적 비율

## 3.2 BIM 품질 세팅

### 📋 파일 위치: `config/bim_quality_config.py`

BIM 품질은 **4가지 지표**로 측정:

| 지표 | 영어 | 의미 | 좋은 값 |
|-----|------|------|--------|
| WD | Warning Density | 경고밀도 (100㎡당 경고 수) | 낮을수록 좋음 |
| CD | Clash Density | 충돌밀도 (100㎡당 충돌 수) | 낮을수록 좋음 |
| AF | Attribute Fill | 속성 채움률 (%) | 높을수록 좋음 |
| PL | Phase Link | 공정 연결률 (%) | 높을수록 좋음 |

### BIM 품질 프리셋

```python
# 우수 (Excellent)
WD = 0.3   # 100㎡당 0.3개 경고
CD = 0.1   # 100㎡당 0.1개 충돌
AF = 0.98  # 98% 속성 채움
PL = 0.95  # 95% 공정 연결
→ 품질 점수: 0.93

# 양호 (Good)
WD = 0.5
CD = 0.2
AF = 0.95
PL = 0.90
→ 품질 점수: 0.86

# 보통 (Average)
WD = 0.8
CD = 0.4
AF = 0.85
PL = 0.75
→ 품질 점수: 0.68

# 미흡 (Poor)
WD = 1.5
CD = 0.7
AF = 0.70
PL = 0.60
→ 품질 점수: 0.44
```

**실무 의미**:
- Excellent: 대형 설계사, 고급 BIM 소프트웨어
- Good: 일반 설계사, 표준 BIM 프로세스
- Average: BIM 도입 초기 단계
- Poor: BIM 미숙, 품질 관리 부족

## 3.4 에이전트 세팅

### 📋 파일 위치: `agents/*.py`

5개 에이전트, 각자 **선호도**와 **영향력** 있음

| 에이전트 | 선호도 | 영향력 | 특징 |
|---------|--------|--------|------|
| 건축주 | 0.25 (빡빡) | 40% | 최종 의사결정권자 |
| 시공사 | 0.75 (여유) | 25% | 작업 여유 확보 선호 |
| 감리사 | 0.50 (중립) | 20% | 계약 조건 기준 |
| 설계사 | 0.40 (약간 빡빡) | 10% | 기술적 해결 가능성 |
| 금융사 | 0.60 (약간 여유) | 5% | 리스크 보수적 평가 |

**선호도 의미**:
- 0.25 = 이슈 범위 중 25% 위치 선호 (빡빡)
- 0.75 = 이슈 범위 중 75% 위치 선호 (여유)

**예시**:
```
이슈: 지연 6~10주
건축주 선호: 6 + (10-6) × 0.25 = 7주 (빡빡하게)
시공사 선호: 6 + (10-6) × 0.75 = 9주 (여유있게)

최종 결정: 가중 평균
= 7×0.40 + 9×0.25 + 8×0.20 + 7.6×0.10 + 8.4×0.05
= 7.83주
```

## 3.5 금융 계산 세팅

### 📋 파일 위치: `models/financial.py`

#### 금리 인상 구조

```python
지연 개월 → 금리 인상
1개월: 0bp (유예)
2개월: 0bp (유예)
3개월: 10bp (0.1%p)
5개월: 30bp (0.3%p)
7개월: 50bp (0.5%p)
10개월: 100bp (1.0%p, 최대)
```

**bp (basis point)**: 0.01%
- 10bp = 0.1%
- 100bp = 1.0%

#### 금융 비용 계산

```python
# 예시: 지연 5개월 (150일)
PF 대출액 = 20억 × 0.7 = 14억
기본 금리 = 5.5%
금리 인상 = 30bp = 0.3%
신규 금리 = 5.5% + 0.3% = 5.8%

# 추가 이자
추가 이자 = 14억 × 0.3% × (150/365)
         = 14억 × 0.003 × 0.411
         = 1,726만원

# 간접비 (일 0.1%)
간접비 = 20억 × 0.1% × 150일
      = 20억 × 0.001 × 150
      = 3억원

# 총 금융 비용
총 금융 비용 = 1,726만원 + 3억원
           = 3.17억원
```

# 4. 계산 방식 상세

## 4.1 이슈 발생 계산

### 발생 확률 계산

```
매일 체크:
  random.random() < occurrence_rate

예시: I-01 (발생률 1.5%)
  시공 단계 300일 동안 매일 체크
  발생 확률 = 1 - (1-0.015)^300 = 99%
  → 거의 확실히 1번은 발생
```

## 4.2 BIM 효과성 계산

### 1단계: 지표 정규화

```python
# WD 정규화 (낮을수록 좋음)
WD_norm = 1 - WD/2.0
예: WD=0.5 → 1 - 0.5/2.0 = 0.75

# CD 정규화 (낮을수록 좋음)
CD_norm = 1 - CD/1.0
예: CD=0.2 → 1 - 0.2/1.0 = 0.80

# AF, PL은 이미 0~1
AF = 0.95
PL = 0.90
```

### 2단계: 이슈별 가중치 적용

```python
# I-01 (설비-구조 간섭) 가중치
weights = {
  'CD': 0.7,  # 충돌밀도 70%
  'WD': 0.2,  # 경고밀도 20%
  'AF': 0.1,  # 속성채움률 10%
  'PL': 0.0   # 공정연결률 0%
}

# 효과성 계산
effectiveness = 0.7×0.80 + 0.2×0.75 + 0.1×0.95 + 0.0×0.90
              = 0.56 + 0.15 + 0.095 + 0.0
              = 0.805 (80.5%)
```

### 3단계: 탐지 확률 계산 (Sigmoid)

```python
# Sigmoid 함수
sigmoid = 1 / (1 + exp(-10 × (0.805 - 0.5)))
        = 1 / (1 + exp(-3.05))
        = 1 / (1 + 0.047)
        = 0.955

# 최종 탐지 확률
detection_prob = base_detectability × sigmoid
               = 0.95 × 0.955
               = 0.907 (90.7%)
```

## 4.3 협상 결과 계산

### 가중 평균 방식

```python
# 이슈: 지연 6~10주, 비용 2.5~4.0%
delay_min = 6, delay_max = 10
cost_min = 0.025, cost_max = 0.04

# 에이전트별 선호도 × 영향력
final_position = Σ (선호도 × 영향력)
               = 0.25×0.40 + 0.75×0.25 + 0.50×0.20
                 + 0.40×0.10 + 0.60×0.05
               = 0.4575 (45.75%)

# 최종 값
agreed_delay = 6 + (10-6) × 0.4575 = 7.83주
agreed_cost = 0.025 + (0.04-0.025) × 0.4575 = 0.0319 (3.19%)
```

## 4.4 BIM 조기 탐지 절감 계산

### BIM OFF (전통 방식)

```python
협상 결과: 7.83주, 3.19%
불확실성 곱: 1.10 (10% 추가)
최종: 8.61주, 3.51%
```

### BIM ON - 조기 탐지 성공

```python
# 1단계: 협상 (탐지됨 반영)
협상 결과: 7.83주, 3.19%

# 2단계: BIM 절감 효과
detection_phase = "설계"
절감률 = 70% (설계 단계 탐지)
품질 보너스 = 0.805 × 15% = 12%
최종 절감률 = 70% + 12% = 82%

# 3단계: 최종 값
actual_delay = 7.83 × (1 - 0.82) = 1.41주
actual_cost = 0.0319 × (1 - 0.82) = 0.0057 (0.57%)
```

**절감 효과**:
- 지연: 8.61주 → 1.41주 (7.2주 절감, 84% 감소)
- 비용: 3.51% → 0.57% (2.94%p 절감, 84% 감소)

# 8. Q&A (예상 질문)

## Q1. 시뮬레이션 결과가 정확한가?

**A**: 업계 벤치마크 대비 검증됨
- McGraw Hill (2014) 기준:
  - Traditional: 22% 예산 초과
  - BIM: 8% 예산 초과
- 본 시스템 결과와 일치
- 다만, 프로젝트 특성에 따라 ±10% 오차 가능

## Q2. 어떤 프로젝트에 적용 가능한가?

**A**: 중소형 건설 프로젝트 (현재)
- 연면적: 500~10,000㎡
- 예산: 10억~100억
- 공기: 6개월~24개월
- 템플릿 추가로 확장 가능

## Q3. BIM 소프트웨어 종류는 상관없나?

**A**: 품질 지표(WD, CD, AF, PL)만 맞으면 됨
- Revit, ArchiCAD, Allplan 모두 가능
- 소프트웨어보다 **품질 관리**가 중요
- 품질 측정 도구 필요 (Solibri, Navisworks 등)

## Q4. 실제 프로젝트 데이터 입력 가능한가?

**A**: 가능함
- 프로젝트 템플릿 추가 (10분 소요)
- 이슈 카드 커스터마이징 (과거 프로젝트 이슈 반영)
- 협상 가중치 조정 (회사 특성 반영)

## Q5. API 키가 필요한가?

**A**: 선택사항
- 있으면: LLM 기반 리얼한 회의록
- 없으면: 템플릿 기반 회의록
- 계산 결과는 동일

## Q6. 시뮬레이션 시간은?

**A**: 약 5분
- BIM OFF: 2분
- BIM ON: 2분
- 그래프 생성: 1분

## Q7. 여러 번 실행하면 결과가 달라지나?

**A**: 약간 다름 (확률 기반)
- 이슈 발생이 확률적
- 10회 평균 내면 안정적
- 비교 모드는 동일 시드 사용 (일관성)

## Q8. 실제 ROI가 이렇게 높나?

**A**: 프로젝트 특성에 따라 다름
- 단순 프로젝트: 200~300%
- 보통 프로젝트: 400~600%
- 복잡 프로젝트: 800~1,200%
- 본 시스템: 보수적 추정 (안전 마진)
