## 🎯 이 프로젝트는 뭘 하는 건가요?

### 간단 설명
"건설 현장에서 BIM(3D 건물 모델)을 쓰면 정말 좋은지" 컴퓨터로 가상 실험을 해봤습니다.

### 어떻게 실험했나요?
1. **가상의 건설 프로젝트 설정**
   - 계획 기간: 360일 (약 1년)
   - 계획 예산: 20.3억원
   - 실제 현장처럼 여러 문제들이 발생하도록 설정

2. **두 가지 방식으로 각각 실험**
   - **방식 1**: BIM 없이 전통적인 2D 도면만 사용

3. **1년 동안 프로젝트 진행 시뮬레이션**
   - 총 22개의 문제가 발생

## 🏗️ 시뮬레이션 구조 (어떻게 작동하나요?)

### 1. 참여자들 (에이전트)

프로젝트에는 5명의 참여자가 있습니다:

```
👔 건축주 (Owner)
   - 역할: 돈을 내는 사람, 빨리 끝내고 싶어함
   - 영향력: 가장 큼 (40-45%)
   - 성격: 비용과 일정을 가장 중요하게 생각

🔨 시공사 (Contractor)
   - 역할: 실제로 짓는 회사
   - 영향력: 두 번째로 큼 (22-25%)
   - 성격: 일정 여유를 원함

📐 설계사 (Designer)
   - 역할: 설계도를 그리는 사람
   - 영향력: 중간 (10%)
   - 특별 능력: BIM을 쓰면 문제를 미리 발견!

👷 감리사 (Supervisor)
   - 역할: 공사 감독, 법규 체크
   - 영향력: 중간 (20%)
   - 성격: 품질과 안전을 최우선

🏦 은행 (Bank)
   - 역할: 돈을 빌려주는 곳
   - 영향력: 적음 (5%)
   - 특별: 프로젝트가 늦어지면 이자율 올림
```

### 2. 프로젝트 단계

공사는 3단계로 진행됩니다:

```
설계 단계 (90일) → 착공 단계 (90일) → 준공 단계 (180일)
```

**BIM의 힘**: BIM을 쓰면 "설계 단계"에서 많은 문제를 미리 발견할 수 있어요!

### 3. 문제 발생과 해결 과정

시뮬레이션에서는 실제 현장처럼 여러 문제가 발생합니다.

#### 예시: "RFI 폭증으로 공정 지연" 문제

**실제 로그에서 발생한 문제:**

**BIM 없을 때 (전통 방식):**
```
문제 발생: 준공 단계 112일째
- 현장에서 갑자기 발견: "설계도에 정보가 부족해요!"
- 월 40-60건의 질문이 쏟아짐
- 결과:
  * 공사 지연: 2.73주 (19일)
  * 추가 비용: 3,881만원 (금융비용)
  * 미리 발견 못함: 늦게 알아서 손해 큼
```

**BIM 쓸 때:**
```
문제 발견: 설계 단계에서 미리 발견!
- BIM 3D 모델로 검토하다가 발견: "정보가 부족할 것 같아요"
- 미리 보완 작업 진행
- 결과:
  * 공사 지연: 0.40주 (3일만) - 86% 감소!
  * 추가 비용: 574만원만 (금융비용) - 85% 감소!
  * 조기 탐지 성공: 손해를 크게 줄임
```

## 💡 왜 BIM이 이렇게 효과적인가?

### BIM의 핵심 기능

실제 코드에서 구현된 BIM의 능력:

```python
# 1. 조기 탐지 (Early Detection)
if issue.detectable_with_bim and current_phase == "설계":
    # 설계 단계에서 문제 발견!
    detection_success = True
    delay_reduction = 0.85  # 지연 85% 감소
    cost_reduction = 0.90   # 비용 90% 감소
```

**3가지 BIM 품질 지표:**

1. **정확도 (Accuracy)**: 0.90
   - 의미: 설계 정보가 90% 정확함
   - 효과: 정보 누락이나 오류가 줄어듦

2. **충돌 검토 (Clash Detection)**: 0.85
   - 의미: 배관, 구조물 간 충돌을 85% 발견
   - 효과: "배관이 겹친다!" 같은 문제를 미리 발견

3. **자동화 (Automation)**: 0.88
   - 의미: 물량 계산 등을 88% 자동으로 처리
   - 효과: 계산 실수가 줄어듦

### BIM 적용 시 변화

```
문제 발생 → BIM 체크 → 68%는 설계 단계에서 발견
                      ↓
                  지연 85% 감소
                  비용 90% 감소
```

## 📊 재무 분석 (돈 계산)

### 비용 구조

**BIM OFF (전통 방식):**
```
계획 예산:        20.3억원
────────────────────────────
직접 비용:        +5.8억원  (추가 인력, 재시공 등)
금융 비용:        +8.4억원  (지연으로 인한 이자)
────────────────────────────
총 비용:          34.5억원
초과액:           14.2억원 ❌
```

**BIM ON:**
```
계획 예산:        20.3억원
BIM 도입 비용:    +0.5억원  (BIM 소프트웨어, 교육)
────────────────────────────
직접 비용:        +2.0억원  (추가 비용 대폭 감소)
금융 비용:        +3.4억원  (지연 적어서 이자 적음)
────────────────────────────
총 비용:          25.7억원
초과액:           5.4억원 ✅
────────────────────────────
전통 방식 대비:   -8.8억원 절감!
```

### 금융 비용 계산 원리

```python
# 실제 코드에서의 금융 비용 계산
daily_interest_rate = annual_rate / 365
delay_cost = project_cost * daily_interest_rate * delay_days
```

**예시:**
- 연이자율: 5.5%
- 일일 이자율: 5.5% ÷ 365 = 0.015%
- 100일 지연 시: 20억 × 0.015% × 100 = 3,000만원 추가

**BIM OFF**: 415일 지연 → 8.4억 금융비용
**BIM ON**: 167일 지연 → 3.4억 금융비용

### 건설 용어

- **에이전트**: 시뮬레이션에서 사람처럼 행동하는 프로그램
- **협상**: 여러 참여자가 의견을 조율하는 과정
